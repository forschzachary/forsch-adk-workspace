ADK vocabulary map

the point: tighten the nouns. A phrase should tell you where code lives, who owns it, how it runs, and what can safely change. Fuzzy words make fuzzy agents. tragic little soup.

source of truth / manifest
runtime loop / ADK machinery
reusable package code
user-facing entrypoint
generation / factory layer
safety / evaluation layer

Use these nouns

agent contractThe canonical promise: identity, package, tools, routes, safety level, smoke prompts. Lives in agent_specs/agents.yaml.
runtime packageThe real Python agent repo. If it changes behavior, tests, or tools for one agent, it belongs under agents/<name>/.
root agentThe exported ADK object. This is what ADK Runner loads: usually root_agent from agent.py.
shared componentReusable tool/client/schema/test helper. If two agents need it, it graduates to components/.
web wrapperA thin adapter so ADK Web can see the real runtime package. It should not become the real agent.
bridge routeThe Discord routing rule: channel/input -> selected ADK agent -> ADK Runner -> response stream.

Avoid these mushy phrases

agent fileToo vague. Say runtime package, root agent, web wrapper, or manifest entry.
the UISay ADK Web surface if it is for testing/editing, or Discord surface if Zach uses it live.
toolsSplit into tool wrapper (ADK callable), client (API code), and credential gateway (Authsome).
configSay agent contract, bridge route, model route, or secret surface. Different risks.
agent factoryGood phrase, but reserve it for generation from a spec. Not hand-editing five places while squinting.
doneOnly means done after contract, package, wrapper, route, docs, tests, and smoke prompt all agree. inconvenient, yes.
01Define contract

Edit or generate agent_specs/agents.yaml. This is the source noun.

02Render package

Create/update agents/<name>/: instructions, root agent, tools, tests, evals.

03Expose surface

Add web_agents/<name>/ for ADK Web and route Discord through bridge/.

04Wire components

Move shared clients and tool wrappers into components/; keep domain behavior in the agent.

05Run gates

Import smoke, unit tests, route tests, eval prompts, service health if relevant.

06Promote

Only after evidence: green checks, clean route, updated docs, and a stability verdict.

short version: The contract names the agent. The runtime package is the agent. The wrappers expose it. The bridge routes to it. Components give it hands. LiteLLM gives it a brain. The Stability Governor decides whether it has earned more autonomy.
